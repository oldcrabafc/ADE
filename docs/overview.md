# ADE Pro 概览

## 项目简介

ADE Pro 是面向审计人员的本地数据导入、清洗、查询与规则分析工具。它以现有 Audit Data Explorer（ADE）为主体，继续使用 PySide6 桌面应用和 DuckDB 本地查询能力，同时吸收 Ledger Analysis 的序时账清洗能力。

系统目标是替代传统的“Excel + 脚本 + 通用数据库工具”组合，让审计人员在本地完成从原始 Excel 到标准账本、查询、规则分析和导出的完整流程。

## 主流程

```text
Excel 原始数据
  -> 字段映射 / 金额规则 / TOML 配置清洗
  -> DuckDB 清洗生成标准 ledger
  -> Parquet 标准数据集包
  -> DuckDB 连接 Parquet 查询
  -> 可视化查询 / SQL 查询 / 审计规则
  -> 导出
```

## 项目背景

常见审计数据处理链路：

- 企业系统导出 Excel 或 CSV
- 审计人员手工整理表头、日期、金额、借贷方向
- Python 脚本或临时工具转换
- 数据库导入
- SQL 查询和规则检查

典型链路：Excel -> Parquet -> DuckDB -> DBeaver。

痛点：

- 工具链割裂，非技术审计人员学习成本高
- Excel 处理大数据、多表关联和复杂查询效率低
- 不同客户字段命名差异大，重复清洗成本高
- 借贷方向、金额口径、账套和公司代码容易混淆
- 缺少审计专用能力，例如异常检测、穿透分析、关键词检索和规则复用

## 项目目标

- 数据导入：支持 Excel/CSV/DuckDB 来源，提供预览、字段映射、配置清洗和导入报告
- 字段标准：采用 Ledger Analysis 字段命名作为 ADE Pro 标准账本字段
- 标准数据集：以 Parquet 固化标准 `ledger` 数据，作为可分享、可复用的数据资产
- 本地查询：以 DuckDB 连接 Parquet 并创建 `ledger` 查询入口
- 兼容迁移：通过 `journal` 兼容视图支持 ADE 现有 SQL、规则和界面
- 数据集复用：下次启动时读取本地数据集清单，不需要重新导入原始 Excel
- 数据集分享：同事可直接打开标准 `ledger` Parquet 数据集包进行查询
- 数据浏览：类 Excel 表格浏览
- 可视化查询：由界面条件生成 SQL
- SQL 查询：编辑、执行、历史、导出
- 审计分析：当前提供大额、周末、摘要为空等基础规则；更多规则留待后续版本

导入提供两种模式：

- 新导入：覆盖该客户该会计年度数据
- 追加导入：保留历史并追加新记录

## 核心设计理念

- Excel 只作为输入，不作为分析存储
- DuckDB 作为核心本地查询和清洗引擎
- Parquet 是标准 `ledger` 数据的固化和交换格式
- `ledger` 是标准账本查询入口
- 客户隔离：一个客户一个数据库文件或目录
- 本地优先，不依赖云端服务
- TOML 配置驱动清洗，减少重复手工映射
- 标准字段统一使用英文 `snake_case`，便于 SQL 输入和跨项目协作

## 数据集体验

启动页或查询页应显示“最近数据集”，便于审计人员直接打开已经转换过的数据：

```text
客户：ABC Company
年度：2024
最近转换：2026-05-12 10:30
行数：123,456
期间：2024-01-01 至 2024-12-31
状态：可查询
```

其中“期间”来自 `ledger.posting_date` 的最早和最晚日期，导入完成后写入数据集 manifest，避免每次启动都扫描 Parquet。

## 适用场景

- 会计师事务所
- 内部审计
- 财务数据分析
- 企业数据检查

ADE Pro 特别适合序时账、总账明细账和审计规则分析。
